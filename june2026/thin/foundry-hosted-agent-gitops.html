Anyone Can Build an AI Agent in a Day. Almost Nobody Can Operate One Reliably.

By Pratik Dhanave
GitOpsMicrosoft FoundryMLOpsDevOpsAgentic AI

That's the core message of a Microsoft Developer Community post on running Foundry Hosted Agents with GitOps and GitHub — and it mirrors what I keep seeing in enterprise AI: the gap isn't in building agents, it's in governing them.

The problem looks like this: prompts get tweaked in a portal text box with no history; tool configs change without review; deployments happen from someone's laptop; and when quality drops, there's no rollback and no audit trail. The fix? Treat your agent like software, not an experiment.

Microsoft's reference approach puts everything that shapes agent behaviour — system prompts, tool definitions, model settings, MCP server configs, safety policies, infrastructure — into a Git repository as the single source of truth. Every change flows through a pull request. GitHub Actions validates, builds, and deploys. Promotion to production requires an explicit approval gate. No human ever deploys by hand.

What stood out to me:

  1. Prompts are code. A "harmless" prompt edit can shift tone, scope, or factual grounding. Versioning prompts as plain text files makes every change reviewable and reversible.
  2. Build once, promote everywhere. The same container image moves from dev → test → prod. No per-environment rebuilds, no drift.
  3. Foundry Toolboxes (preview) fix tool sprawl. Define tools once in a versioned, centrally governed bundle exposed through one stable MCP endpoint. Update the toolbox, and every agent picks it up — no redeployment.
  4. Your issues are your audit trail. Linking GitHub Issues to prompt changes, model upgrades, and rollbacks means a reviewer can trace any production deployment back to the original intent behind it.

The biggest pitfall called out: skipping structured evaluation before promotion. That's how regressions reach production. As agents move from demos to systems that touch real data and real customers, delivery discipline becomes the differentiator. GitOps isn't new — applying it rigorously to AI agents is.

Source: "Building and Operating a Microsoft Foundry Hosted Agent with GitOps and GitHub," Microsoft Azure Developer Community blog (2026).
