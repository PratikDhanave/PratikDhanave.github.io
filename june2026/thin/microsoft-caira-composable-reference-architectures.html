CAIRA: We're Moving from Frameworks Humans Adopt to References Agents Compose

By Pratik Dhanave
CAIRAAzure AI FoundryAgentic AIIaCOpen Source

Microsoft just open-sourced something that quietly reframes how we build enterprise AI: CAIRA — Composable AI Reference Architectures. What makes it interesting isn't another framework to adopt. It's the delivery model.

CAIRA isn't a library you import or a template you fork and pray over. It's a reference library designed for your coding agent to read: you point your agent (Copilot, etc.) at the repo and it copies or adapts only the pieces that fit your scenario. You describe what you want — say, "an agentic monitoring system to detect security issues in a GitHub repo using Foundry, an API, and a React dashboard" — and the agent assembles it from vetted components.

What's inside:

  • Terraform (infrastructure-as-code) for Azure AI Foundry, with Basic and Standard variants and private-networking options
  • Agentic API references in TypeScript (OpenAI Agents SDK & Foundry Agent Service) and C# (Microsoft Agent Framework)
  • A minimal React frontend with a BFF proxy

Each piece is independent — easy to read, validate, copy, and modify. No monolith to untangle. It also ships with a CAIRA Assistant chat mode that walks through deployment, validates prerequisites, and helps verify resources afterward — so architecture selection becomes a conversation rather than a manual hunt.

The bigger signal here: reusable building blocks that an AI assembles into your stack, with you approving each step. Worth a look if you're scaling agentic AI in production.
